  # Feature - Promises

 

  - Fun Fact: Node adopted Promises in 2009,

    but removed them in 2010

 

  - Deno's API's are built for asnyc/await,

    for example:

 

    readFile(name: string): Promise<Uint8Array>

 

 

  // API Reference:

  // deno.land/typedoc/index.html

  # Feature - Modules

 

  - No `require` function.

  - ES modules work out of the box.

 

  // For example:

 

  // in hello.ts

  export function hello(name: string) {

      console.log(`Hello, ${name}!`);

  }

 

  // in world.ts

  import { hello } from "./hello.ts";

  hello("World");

  # Feature - Package Management

 

  - No NPM, no `node_modules`, no `index.js`.

  - Import modules from URLs - like a browser!

  - Modules are cached in `$DENO_DIR`.

  - Import maps are supported. No SRI yet.

  - Standard library:

 

  import * as fs from "https://deno.land/

      std@v0.11.0/fs/mod.ts";

 

  - Registry with userland modules:

 

  import * as pg from "https://deno.land/

      x/postgres@v0.2.0/mod.ts";

  # Feature - Web Compatibility

 

  "The subset of Deno programs which are

   written completely in JavaScript and do

   not use the global `Deno` namespace, ought

   to also be able to be run in a modern web

   browser without change."

       -- Deno Manual

 

  - Unlike in Node, `window` is defined.

  - For example, `fetch` just works!

  - Compatibility is WIP, see: deno.dev/wpt

  # Feature - Dev Tools

 

  - `deno fmt`: Format code consistently

  - `deno bundle`: Bundle code as single file

  - `deno info`: Show module dependency tree

  # Getting Started

 

  - Visit the website: deno.land

  - Install Deno:

 

$ brew install deno   # On MacOS

$ scoop install deno  # On Windows

 

  - Run your first script:

 

$ deno https://deno.land/welcome.ts
